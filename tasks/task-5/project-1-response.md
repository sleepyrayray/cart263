# Project 1 Responses

by Ray Hernaez

## 1- Snake Game
**Link:** https://yaxuanpang.github.io/cart263/topics/project1/

This project instantly reminded me of the classic Snake game from older phones, which made it feel familiar in a good way. The gameplay is pretty straightforward: eat the “good” stuff, avoid the “bad” stuff, and don’t crash into the edges. One detail I really liked was the **5 lives**, shown as **hearts in the corner**. It’s a simple UI choice, but it makes the game feel more forgiving and readable. The **score counter** is also helpful, although hitting **50 points** feels like it may take a little while, so the win condition can start to feel a bit grindy. A cool twist is how the avoidable food starts turning into more obstacles over time, which makes the challenge ramp up naturally.

The game experience and vibe change a lot as you play. At the start it feels relaxed, but once the snake grows, the stress increases fast and the game gets intense. Visually, it has a nice retro feel. Its bright, vibrant colours and simple shapes make it clean and readable. On the code side, I noticed they used **OOP** to organize different game elements, which helps keep things structured. However I saw a file called `bird.js`, but in the actual game I mostly noticed circles and squares, so it was hard to connect that “bird” idea visually. I think stronger visual representation and maybe some simple sound effects would make the experience even more satisfying.

## 2- Aquarium
**Link:** https://philippe-bn.github.io/cart263/projects/project-1/

This one was excellent—! It's simple, but *really* effective. The most satisfying part is how the fish **group up and follow the mouse cursor**. It feels playful and responsive, and it’s the kind of interaction that’s instantly fun without needing complicated instructions. The colours are super vibrant, and the canvas being large (almost full screen) makes it feel like you’re actually staring into a big aquarium window. I also liked that the fish have different colours, because it helps you see the “schooling” behavior more clearly when they start grouping up together.

A small design detail I enjoyed is that the fish have a short attention span. When you stop moving the mouse, they lose interest pretty quickly. That makes them feel more real and alive, like they’re reacting to you, but not in a robotic way. In terms of code, this also uses **OOP** to organize the elements, which makes sense for something like multiple fish objects moving around. The comments were helpful overall, though I noticed there was a fair amount of unused/commented-out code left behind. That’s not necessarily “bad” (maybe they’re saving it for future changes), but cleaning it up could make the project easier to read and maintain.

## 3- Puzzle
**Link:** https://dianaedvi.github.io/CART-263/projects/project_1/

This project felt like a successful puzzle simulation and honestly reminded me of those captcha-style puzzles where the user prove they're human. I liked that there are **four different puzzle blocks**, and they **randomize each time you reload**, which keeps it from feeling repetitive. Solving it made me use my brain a lot, but in a fun way especially because you’re working toward revealing a hidden image once everything is solved. The background pattern with ducks adds a silly, cartoony vibe that keeps the whole thing light and not too serious.

From a code perspective, the main issue is that the script is pretty long because it’s all in one file. It works, but it becomes harder to read and manage. Breaking it into classes (OOP) or separate modules would probably make it cleaner and easier to expand. Also, the layout doesn’t adapt well when the browser window gets smaller... On a narrow screen it starts to feel awkward and you kind of need a wide browser to properly see and play the project.

---

## Conclusion
All three projects were engaging, but in different ways. The Snake game builds tension through increasing difficulty, the Aquarium feels satisfying because of smooth interaction and motion, and the Puzzle makes you think while rewarding you with an image reveal. Code-wise, the projects that used **OOP** felt easier to follow conceptually (especially when there are lots of moving parts like multiple elements on screen). The biggest takeaway for me is how much small choices matter, like clear UI, responsive motion, and replay value. Even simple ideas become memorable when the interaction and presentation feel intentional. Nice job, everyone.
